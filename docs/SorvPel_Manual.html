1. Apresentação do Sistema

O SorvPel Controle de Estoque é um sistema completo para registro e acompanhamento de entradas e saídas de insumos e produtos em fábricas de sorvetes e açaí. O sistema é composto por dois módulos principais:

  • Portal Web — interface de gerenciamento acessada via navegador, com dashboard, relatórios, exportação CSV e área administrativa.
  • Terminal ESP32 — dispositivo físico instalado no chão de fábrica; os operadores registram movimentos de estoque diretamente pelo teclado matricial, sem precisar de computador.

Funcionalidades Principais

FuncionalidadeDescrição
Registro de movimentosEntradas e saídas por produto, operador e turno
Dashboard em tempo realKPIs do dia, estoque por categoria (drilldown), gráfico e últimos registros

Gestão de operadoresCadastro com crachá e PIN de acesso ao terminal
Gestão de produtosSKU numérico, nome, unidade, categoria

Múltiplos usuários webPerfis admin e usuário comum

4. Instalação do Servidor

4.1 Windows — Instalador Automático (recomendado)

1
Baixe ou clone o repositório do projeto. 
git clone https://github.com/EddyAlvess/ControleEstoque.git
cd ControleEstoque
2
Instale o Docker Desktop e certifique-se de que ele está em execução (ícone na bandeja do sistema).
3
Execute o instalador com botão direito → "Executar com PowerShell": 
install\windows\SorvPel_Setup.ps1
O assistente irá guiá-lo pelos passos de configuração (porta, senhas, chave API) e instalação automática.
4
Ao concluir, clique em "Abrir no Navegador" para acessar o portal.

4.2 Linux — Instalador Automático

1
Clone o repositório: 
git clone https://github.com/EddyAlvess/ControleEstoque.git
cd ControleEstoque
2
Execute o instalador como root: 
chmod +x install/linux/install.sh
sudo bash install/linux/install.sh
O script detecta a distribuição, instala o Docker se necessário, configura as senhas e sobe os serviços.

4.3 Configuração Manual (avançado)

Se preferir configurar manualmente, copie o arquivo de exemplo e edite:

cp .env.example .env

5.1 Conexão de Hardware

ℹ️ Endereço I2C O endereço padrão do LCD I2C é 0x27. Caso seu display seja diferente (ex: 0x3F), ajuste LCD_I2C_ADDR no config.h.

5.2 Gravação do Firmware — Windows

1
Conecte o ESP32 ao computador via cabo USB.
2
Execute o gravador: 
install\esp32\flash_windows.ps1
A janela detectará a porta COM automaticamente.
3
Preencha os campos: SSID, senha WiFi, URL do servidor, Chave API ESP32 e ID do terminal.
4
Clique em ▶ Gravar Firmware. Aguarde ~1 minuto.

5.3 Gravação do Firmware — Linux

chmod +x install/esp32/flash_linux.sh
./install/esp32/flash_linux.sh

O script detecta a porta USB automaticamente (/dev/ttyUSB0 ou /dev/ttyACM0), solicita as configurações e grava o firmware.

5.4 Verificação Pós-gravação

Após a gravação, o ESP32 exibirá no LCD:

Conectando WiFi...
SSID: SuaRede

Aguarde...

Se a conexão for bem-sucedida:

SorvPel Estoque
--------------------

       iniciar
⚠️ Problema de conexão WiFi Verifique se o SSID e senha foram digitados corretamente. O ESP32 suporta apenas redes 2.4 GHz (não 5 GHz). Certifique-se de que o servidor está acessível pela rede local.

6. Primeiro Acesso e Configuração Inicial

1
Abra o navegador e acesse http://<IP-DO-SERVIDOR>:<PORTA>
(ex: http://192.168.1.100:8080 ou http://localhost:8080)
2
Faça login com as credenciais padrão:
Usuário: admin  |  Senha: (definida durante a instalação)
3
Altere imediatamente a senha padrão: clique no ícone 🔑 no rodapé do menu lateral.
4
Configure os dados básicos na ordem sugerida:
  1. Admin → Categorias: crie as categorias de produtos (ex: Açaí, Sorvete, Picolé)
  2. Admin → Produtos: cadastre os produtos com código SKU numérico
  3. Admin → Operadores: cadastre os operadores com crachá e PIN
  4. Admin → Turnos: configure os turnos de trabalho (hora início/fim)
  5. Admin → Usuários Web: crie contas para supervisores/gestores
⚠️ SKU numérico obrigatório O teclado matricial do ESP32 possui apenas teclas numéricas (0-9). Os códigos SKU dos produtos devem ser configurados como números (ex: 1, 2, 101, 202) para que os operadores possam localizá-los no terminal.

7. Operação — Portal Web

7.1 Dashboard

O dashboard exibe uma visão consolidada da operação do dia:

SeçãoDescrição
KPIs do DiaTotal de entradas, saídas e saldo líquido do dia corrente
Estoque por CategoriaCards com saldo acumulado por categoria; clique para ver detalhamento por produto
Gráfico de HojeBarras comparando entradas x saídas por produto no dia
Últimos RegistrosOs 10 movimentos mais recentes

7.2 Relatórios

Acesse em Relatórios no menu lateral. Filtros disponíveis:

  • Tipo: Entrada, Saída ou todos
  • De / Até: filtro por data e hora (datetime)
  • Turno: filtra pelo turno de trabalho
  • Operador: filtra por operador específico
  • Categoria + Produto: produto sensitizado pela categoria selecionada
  • Itens por página: 10, 50, 100 ou 500 registros

Use "Filtrar" para aplicar os filtros e "✕" para limpar. Clique em "Exportar CSV" para baixar os dados filtrados.

7.3 Administração

Usuários Web

Gerencia contas de acesso ao portal. Perfis disponíveis:

  • admin — acesso total: dashboard, relatórios e toda a administração
  • user — acesso ao dashboard e relatórios apenas

Operadores

Funcionários que operam os terminais ESP32:

  • Nome: nome completo do operador
  • Código do Crachá: código digitado/escaneado para selecionar o operador na lista do terminal
  • PIN do Terminal: senha numérica (mín. 4 dígitos) exigida após selecionar o operador. Badge Cadastrado ou Sem PIN

Produtos

  • SKU: código numérico único do produto (digitado no terminal)
  • Nome: descrição do produto
  • Unidade: unidade de medida (L, kg, un, cx…)
  • Categoria: agrupamento para filtros e dashboard

Categorias

Agrupamentos de produtos (ex: Açaí, Sorvete, Picolé, Outros). Excluir uma categoria desvincula os produtos sem apagá-los.

Turnos

Turnos de trabalho com hora de início e fim. A detecção é automática: ao registrar um movimento, o servidor verifica o horário e associa o turno correto.

Turnos com virada de meia-noite Configure início: 22:00 e fim: 06:00 para turnos noturnos que cruzam a meia-noite.

7.4 Alterar Senha (usuário)

Qualquer usuário pode alterar sua própria senha clicando no ícone 🔑 no rodapé do menu lateral (ao lado do nome do usuário). Informe a senha atual e a nova senha (mínimo 6 caracteres).

8.3 Timeout Automático

Se o terminal ficar inativo por 60 segundos (configurável em MENU_IDLE_MS no config.h), ele retorna automaticamente à tela inicial, cancelando o movimento em andamento.

8.4 Simulador (para testes)

O simulador Python replica o terminal no computador, útil para testes sem hardware físico:

# Iniciar com Docker:
docker compose --profile sim up esp32-sim

Teclas do simulador: as mesmas do teclado real + R recarregar dados, S simular scanner, Q sair.

9. Manutenção

9.1 Iniciar e Parar o Sistema

# Iniciar todos os serviços
docker compose up -d

# Parar todos os serviços (dados preservados)
docker compose down

# Reiniciar apenas o backend
docker compose restart backend

# Ver status
docker compose ps

9.2 Visualizar Logs

# Logs em tempo real (todos os serviços)
docker compose logs -f

# Apenas o backend
docker compose logs -f backend

# Apenas o banco de dados
docker compose logs -f postgres

9.3 Backup do Banco de Dados

# Exportar dump completo
docker compose exec postgres pg_dump -U sorv_user controle_estoque > backup_$(date +%Y%m%d).sql

# Restaurar
docker compose exec -T postgres psql -U sorv_user controle_estoque < backup_20260101.sql
⚠️ Backup regular Recomenda-se backup diário automático via cron (Linux) ou Agendador de Tarefas (Windows). O arquivo .env também deve ser salvo separadamente pois contém as chaves de segurança.

9.4 Atualização do Sistema

# Obter atualizações do repositório
git pull origin master

# Reconstruir e reiniciar containers
docker compose up -d --build

# Executar novas migrações (se houver)
docker compose run --rm backend alembic upgrade head

9.5 Atualização do Firmware ESP32 (OTA)

Com o ESP32 em execução e conectado à rede, é possível atualizar o firmware sem cabo USB:

# Upload OTA pelo IP do dispositivo
cd esp32
pio run -t upload --upload-port 192.168.1.XXX

O IP atual do ESP32 é exibido no monitor serial ao iniciar. Verifique também no roteador WiFi.

9.6 Resolução de Problemas

SintomaCausa provávelSolução
Portal não abre no navegadorContainers não iniciados ou porta bloqueadadocker compose up -d; verificar firewall
"Usuário ou senha incorretos" no loginCredenciais erradasVerifique usuário/senha; redefina pelo banco se necessário
ESP32 não conecta ao WiFiSSID/senha errados ou rede 5 GHzVerificar config.h; o ESP32 só funciona em 2.4 GHz
ESP32 exibe "Erro: HTTP 401"Chave API incorretaVerificar API_KEY em config.h vs ESP32_API_KEY no .env
ESP32 exibe "PIN não cadastrado"Operador sem PIN definidoAdmin → Operadores → Editar → definir PIN
Produto não encontrado pelo códigoSKU incorreto ou produto inativoVerificar SKU em Admin → Produtos; confirmar que está ativo
Banco de dados não iniciaSenha incorreta no .envVerificar POSTGRES_PASSWORD; docker compose down -v para resetar volume
Containers não iniciam após updateMigração pendentedocker compose run --rm backend alembic upgrade head

9.7 Redefinir Senha Admin (emergência)

# Conectar ao banco e alterar a senha diretamente:
docker compose exec postgres psql -U sorv_user -d controle_estoque

# No prompt psql:
UPDATE users SET hashed_password = crypt('nova_senha', gen_salt('bf')) WHERE username = 'admin';
\q

Ou via Python (mais seguro):

docker compose run --rm backend python -c "
from app.services.auth_service import hash_password
print(hash_password('nova_senha'))
"
# Copie o hash e use:
docker compose exec postgres psql -U sorv_user -d controle_estoque \
  -c \"UPDATE users SET hashed_password='HASH_ACIMA' WHERE username='admin'\"
